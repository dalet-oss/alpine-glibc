# alpine-glibc

This repo provides Github [releases](https://github.com/dalet-oss/alpine-glibc/releases) for Alpine Linux `glibc`
packages, for both `amd64` and `arm64` CPU architectures.

Although the artifacts (and intermediate tarballs) are store in Git LFS, please do *not* pull these unless you are a
maintainer of this repo, as Github's LFS bandwidth limitation of 10GB per month would soon be exhausted.  There is no
such limit for downloading artifacts from releases.

## Consuming the released artifacts

Here is an **excerpt** from a multi-architecture `Dockerfile`, showing how you might consume the binary artifacts in
our [releases](https://github.com/dalet-oss/alpine-glibc/releases).

The derivation of this, in turn, came from a [long-since-removed AdoptOpenJDK Dockerfile definition](https://github.com/AdoptOpenJDK/openjdk-docker/blob/master/16/jre/alpine/Dockerfile.openj9.releases.full)
for building an OpenJ9 Alpine Linux Java base image.

```dockerfile
ENV GLIBC_VERSION='2.42'

RUN echo 'Installing glibc-based standard C libraries' && \
    curl -LfsS https://github.com/dalet-oss/alpine-glibc/releases/download/${GLIBC_VERSION}-${TARGETARCH}/glibc-${GLIBC_VERSION}.apk -o /tmp/glibc-${GLIBC_VERSION}.apk && \
    curl -LfsS https://github.com/dalet-oss/alpine-glibc/releases/download/${GLIBC_VERSION}-${TARGETARCH}/glibc-bin-${GLIBC_VERSION}.apk -o /tmp/glibc-bin-${GLIBC_VERSION}.apk && \
    curl -LfsS https://github.com/dalet-oss/alpine-glibc/releases/download/${GLIBC_VERSION}-${TARGETARCH}/glibc-i18n-${GLIBC_VERSION}.apk -o /tmp/glibc-i18n-${GLIBC_VERSION}.apk && \
    curl -LfsS https://raw.githubusercontent.com/dalet-oss/alpine-glibc/refs/heads/master/alpine-packaging-rsa.pub -o /etc/apk/keys/alpine-packaging-rsa.pub && \
    apk add --force-overwrite --no-cache /tmp/glibc-${GLIBC_VERSION}.apk && \
    apk add --no-cache /tmp/glibc-bin-${GLIBC_VERSION}.apk && \
    apk add --no-cache /tmp/glibc-i18n-${GLIBC_VERSION}.apk && \
    rm -rf /tmp/*.apk

# N.B. If you want to use this glibc installation to run a glibc-based JVM, you'll want to install `libgcc`,
#      `libstdc++`, and `zlib` also. Arch Linux is a good source for these binaries, which at the time of writing seem
#      to work just fine when unpacked to the /usr/glibc-compat/lib directory.
```

##  Adding a new glibc version

1. Edit [glibc-versions.yml](glibc-versions.yml) to add an entry for the new glibc version, with dummy checksums.
   Anyone can raise a PR for this!
2. Push this change (e.g. merge the PR) to `master`.
3. Github Actions will launch the `compile-glibc` workflow to compile the new glibc version for `amd64`, and raise a PR
  delivering it and updating the corresponding checksum in [glibc-versions.yml](glibc-versions.yml).
4. When the autogenerated PR is ready, you should review it, and merge if appropriate.
5. Repeat steps (3) and (4) for the `arm64` variant.
6. Github Actions will then launch both of:
    - the `compile-glibc` workflow again (validating checksums); and
    - the `package-apks` workflow to package glibc into appropriate, signed APKs, raising separate PRs for each of the
      `amd64` and `arm64` variants.
7. When the autogenerated PRs are ready, you should review them, and merge if appropriate - merging first the `amd64` PR
   and then the `arm64` PR.
8. Github Actions will then launch the `publish-releases` workflow to publish the new APK sets to our Github
   [releases](https://github.com/dalet-oss/alpine-glibc/releases) page.
9. 💃 🪩 🕺
